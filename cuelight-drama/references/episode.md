# Episode

```bash
cuelight-cli episode create <projectId> --title "第一集" --number 1 --summary "分集大纲" --json
cuelight-cli episode set-outline <episodeId> --file ./.cuelight/<projectId>/episodes/episode-1-outline.txt --json
cuelight-cli episode set-script <episodeId> --file ./.cuelight/<projectId>/episodes/episode-1-script.txt --json
cuelight-cli episode status <episodeId> --json
```

## Outline

分集大纲用于指导剧本文本和分镜，不是营销简介。

建议按以下节奏写：

- 开场钩子：第一屏/第一场立刻有冲突、疑问或强情绪。
- 本集目标：本集主角要争取、隐藏、逃离或揭露什么。
- 阻力升级：对手、环境、误会或规则如何加压。
- 信息/反转：本集至少一个新信息、关系变化或局势反转。
- 情绪落点：观众在本集末尾应感到爽、痛、甜、怕或想继续看。
- 结尾钩子：下一集必须追看的悬念。

不要只写“主角经历困难并成长”这类不可拍摘要。

每集必须有清楚边界：本集只拍本集事件。后续集内容可以作为最后一个钩子出现，但不能进入本集主体动作。若第 1 集正文只写出租车监控，就不能在第 1 集 storyboard 里提前进入第 2 集集贸市场或第 3 集老宅。

## Script / Content

剧本正文要可拍摄，至少包含：

- 场景标题或场景转换。
- 人物动作与调度。
- 角色对白。
- 关键道具使用方式。
- 道具、手机、监控、聊天记录、票据等屏幕信息。
- 声音触发点，例如手机提示音、门声、脚步声、环境声或一句明确对白。
- 情绪变化和停顿。
- 结尾钩子。

## 剧本容量与可分镜性

写 storyboard 前先检查正文容量。正文不足时先扩写剧本，不要让分镜替剧本补窟窿。

- 90 秒单集建议有 5 个 beat；每个 beat 至少给出一个可拍动作、一个情绪变化或一个信息增量。
- 90 秒单集正文建议包含 12-20 个可拍动作/反应/道具操作，6-12 条有效对白或声音触发点，8-14 个短段落。
- 每个短段落只承担一个明确拍摄动作或信息点，方便自然拆成 6-9 条 storyboard item。
- 正文不能只写概述句，例如“王家人继续讨论怎么害她”。必须落成可拍台词、动作、道具或屏幕信息，例如监控里某人说出具体算计、主角截图保存证据、聊天框弹出转账要求。
- 如果正文无法自然拆出目标时长对应的 storyboard item，先补正文动作、对白、屏幕信息或情绪停顿，再写分镜。
- 分镜必须能追溯到正文中的动作、对白、道具、屏幕信息或声音触发；只有结尾钩子可以少量引出下一集。

## 三位资深编剧自检

写完 outline 和 script 后，先模拟三位资深编剧审稿。发现问题时先修剧本，再生成 storyboard。

- 结构编剧：检查开场钩子、本集目标、阻力升级、信息/反转、情绪落点、结尾钩子是否完整；检查是否存在集内容量不足或越集补戏。
- 人物编剧：检查主角是否有主动选择和具体行动；检查情绪是否通过动作、停顿、道具使用、眼神、呼吸、手部动作外化，而不是只写“很难过”“很愤怒”。
- 类型编剧：检查短剧爽点、反击点、危机前兆和信息爆点是否集中；检查是否有可拍的狠台词、证据动作或局势反转。

自检通过标准：本集 90 秒可以独立拍完；主角有明确行动；每个 beat 都有可拍信息；分镜不需要借下一集内容补时长。

## 推荐格式

每集正文推荐按“场景/时间/人物 + 动作行 + 对白 + 道具/屏幕信息 + 声音/停顿”组织。可用传统剧本格式，也可用短剧段落格式，但必须能直接提取场景、角色、道具、对白和动作。

短剧段落示例：

```text
【第1集 监控里的背叛】

场景：出租车后排 / 夜
人物：田雨

田雨坐在后排左侧，手机屏幕停在婚房客厅监控画面。她右手拇指悬在录屏按钮上，车窗外路灯从她肩后滑过。

手机里传来王某压低的声音：“等她死在高温里，房子自然就是我们的。”

田雨没有哭。她慢慢按下录屏，另一只手从包里抽出不孕检查报告，报告纸角在膝前轻轻抖了一下。
```

传统剧本可以使用：

```text
1. 内景 / 赵府寿安堂 / 日
人物：赵阿萤、赵宛瑜

△ 赵阿萤站在屏风旁，右手按着袖中的玉佩。
赵宛瑜：你以为今天还能躲过去？
赵阿萤：（压住呼吸）我从没想躲。
```

短视频/教学稿可以使用时间段、画面、台词、音效结构，但写分镜时要按 `references/storyboard.md` 的时间码规则拆分。

## 验收

- `summary` 或 outline 非空。
- `content` 或 script 非空。
- 第一集能提取至少 1 个可绑定场景、1 个主要角色。
- 90 秒正文能自然拆成 6-9 条 storyboard item，且不需要借后续集内容补时长。
- 每个 storyboard item 都能追溯到正文中的动作、对白、道具、屏幕信息或声音触发。
- 三位资深编剧自检无阻断问题：结构完整、人物有主动行动、类型爽点集中。
- 不因写剧本文本而提交图片、视频或语音任务。
