# 中文好莱坞式剧本正文格式

用于 `episodes/ep-xxx.md` 的 `## 剧本正文` 部分。目标是让 Markdown 中的剧本正文保持稳定缩进，同时服务短剧的钩子、节奏和情绪兑现。

## 基本规则

- 使用 `fountain` 代码块包裹正文，避免 Markdown 吞掉缩进。
- 场景标题顶格：`INT. 地点 - 时间` 或 `EXT. 地点 - 时间`。
- 动作段顶格，写可见动作、画面、声音和可拍行为；不要写抽象心理解释。
- 角色名约 20 个前导空格；同一项目内保持一致。
- 对白约 10 个前导空格；短句优先，保留潜台词和角色语言指纹。
- 括注介于角色名和对白之间，只在动作无法表达时使用，例如 `(压低声音)`。
- 转场可选；短剧默认少用。需要时可用 `CUT TO:`、`MATCH CUT:`。
- 集尾卡点可以放在代码块后单独写，便于质量审查。

## 骨架

````markdown
## 剧本正文

```fountain
INT. 酒店宴会厅 - 夜

宾客围成一圈。林晚站在中央，礼服肩带被红酒浸湿。

                    王曼
          你不是说自己是林家大小姐吗？

林晚没有抬头，只把邀请函翻到背面。

                    林晚
          他们不认我没关系。

门口传来骚动。林氏集团董事长停在灯下。

                    林晚
          我认他们就行。
```
````

## 质量检查

- 是否存在 `fountain` 代码块。
- 场景标题是否顶格，并使用 `INT.` 或 `EXT.`。
- 动作段是否顶格，且只写可见内容。
- 角色名和对白缩进是否全篇一致。
- 对白是否短、带目的、少解释。
- 括注是否克制，没有替代动作描写。
- 结尾是否保留未解决问题，而不是自然收尾。
