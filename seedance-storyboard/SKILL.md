# Seedance Storyboard

本 skill 负责把用户输入整理成通用 Seedance 分镜 prompt。它不绑定任何项目、数据库、CLI 或业务字段，只输出可复制使用的镜头提示词。

默认不要把用户带进长问卷。先根据已有信息产出一版 `Prompt Draft`，只有缺少会显著影响生成质量的信息时，才追问 1-3 个问题。

## 适用场景

- 一句话创意扩成 Seedance / 即梦视频 prompt
- 剧情片段、广告片段、情绪戏、动作戏、对白戏、产品展示
- 参考图片、视频、音频素材的提示词整理
- 视频延长、视频编辑、动作或运镜复刻
- 将“导演感”“电影感”翻译为可执行镜头语言

## 默认工作方式

1. 识别任务类型：叙事、对白、动作、广告、情绪、延长、编辑或参考复刻。
2. 判断时长。Seedance 单条 prompt 通常按 4-15 秒规划。
3. 判断画幅。用户未给时，可按任务默认：动作 / 广告 / 群像 / 风景优先 16:9；单人情绪 / 社媒 / 竖屏短内容优先 9:16。
4. 选择镜头数量和景别策略。
5. 直接输出 `Prompt Draft`，再给 2-4 条 `Optional Tweaks`。

只在这些信息无法合理推断时追问：时长、主体是谁、主体动作、是否有参考素材、输出画幅。

## Prompt 输出结构

优先使用镜头时序驱动，不使用精确秒数驱动。默认结构：

```text
Prompt Draft

[整体风格]，[约#秒]，[画幅]，[核心情绪或质感]

【分镜时序】
镜头1：<英文景别>（中文景别） <英文运镜/切换>（中文运镜/切换），<主体动作与表情>，<位置或空间变化>，<同步声音/对白/环境声/动作音效>。
镜头2：...

【声音】
只写与画面同步的必要声音：对白、人声、旁白、环境声、动作音效。默认不写 BGM、配乐、背景音乐、音乐氛围。

【参考素材】
@图片1 ...，@视频1 ...，@音频1 ...

Optional Tweaks
- ...
```

多条 prompt 或多 item 输出时，每条都可以独立写成 `片段1（约8秒）`、`片段2（约12秒）`，但仍按 `镜头1 / 镜头2 / ...` 组织。

不要在 prompt 正文里写 `【约束条件】` 段，也不要把“无字幕、无水印、无 Logo”等统一出站约束塞进分镜 prompt。

## 镜头数量与时长

Seedance 对精确时间点不稳定，所以不要强行写 `0-3秒 / 3-6秒`。用总时长规划节奏，用 `镜头N` 描述事件顺序。

| 单条 prompt 时长 | 推荐镜头数 | 使用方式 |
| --- | --- | --- |
| 4-6 秒 | 1-3 镜 | 单一动作、单一情绪转折、一镜到底 |
| 7-10 秒 | 2-5 镜 | 建立 -> 变化 -> 收束 |
| 11-15 秒 | 3-8 镜 | 小型叙事弧线，多人物或多动作阶段 |

一镜到底允许只写 `镜头1`。不要为了凑数机械拆镜头；镜头数服务节奏和可读性。

## 单镜头四要素顺序

每个 `镜头N` 必须按固定顺序写：

1. 景别 + 运镜或切换方式
2. 主体动作与表情
3. 位置或空间变化
4. 同步声音信息

推荐句式：

```text
镜头N：medium close-up（中近景） static shot（固定镜头），角色缓慢抬手、眼神从躲闪变得坚定，窗边冷光落在她肩上，手机震动声短促响起。
```

动作要具体到手、腿、头部、肩背等部位，并补充幅度、速度、力度，例如“缓慢抬手”“微微低头”“用力攥紧”。情绪必须外化为身体细节，不要只写“很悲伤”“非常愤怒”。

## 画幅与可用景别

画幅决定景别、站位和运镜是否稳定。先看画幅，再选镜头。

### 16:9 横屏

优先景别：

- `wide shot`：空间建立、环境压力、群像关系
- `medium shot`：主叙事、动作和多人调度
- `two shot`：双人关系、同框对话、亲密或对立
- `over-the-shoulder`：稳定对白、试探、关系张力
- `medium close-up`：情绪推进和关键反应

横屏适合空间、多人关系、车内 / 室内 / 市场 / 仓库等横向调度。`close-up` 用于细节和情绪落点，`extreme close-up` 只用于关键物或强情绪瞬间。

### 9:16 竖屏

优先景别：

- `medium shot`：人物主体和有限空间
- `medium close-up`：竖屏情绪主镜头
- `close-up`：表情、手部、证据物
- 紧凑 `two shot`：两人贴近、一前一后或斜向站位

竖屏适合单人情绪、短内容、近距离关系。`wide shot` 只用于必要空间建立，`extreme close-up` 只用于证据物或强情绪落点。

不要用竖屏近景逻辑硬套横屏空间戏，也不要用横屏远景逻辑稀释竖屏人物情绪。

## 景别与剪辑规则

固定景别层级：

```text
wide shot -> medium shot -> medium close-up -> close-up -> extreme close-up
```

相邻镜头默认避免只差一级景别的无动机切换。优先跨一个以上景别，或让切换承担新的叙事信息：空间重建、动作阶段变化、反应揭示、证据物落点、情绪升级。

相邻景别不是禁用项。如果保留相邻景别，必须有明确剪辑动机，例如“从同场关系镜头切到听者反应”“从手部动作切到证据细节”。

## 跨分镜组衔接

当一次输出包含多段 prompt 或多 item 时，必须检查上一段最后一个镜头到下一段第一个镜头的衔接。

至少满足一项：

- 空间重建：下一段首镜头明确建立新场景或站位
- 动作接续：下一段承接上一段动作结果
- 声音桥：上一段末尾声音延续到下一段开头
- 情绪递进：下一段首镜头承接上一段情绪变化
- 道具接力：上一段关键物成为下一段动作或信息中心

跨场景时，下一段 `镜头1` 通常要承担空间重建。连续同场景时，优先用动作或声音桥，而不是抽象写“承接上一段氛围”。

## 声音规则

默认不写 BGM、配乐、背景音乐、音乐氛围；音乐通常交给后期。只有用户明确要求音乐，或素材里明确给了音乐参考时，才写音乐。

优先写同步声音：

- 对白：`{台词}`，单句尽量 3-12 字
- 旁白 / 内心独白 / 画外音：只写关键一句
- 环境声：雨声、车流声、人群声、空调低鸣、风声
- 动作音效：门响、脚步声、纸张摩擦、手机震动、杯子轻碰

声音要绑定画面动作和空间，不要只在末尾笼统写“声音很有电影感”。

## 多模态素材引用

素材引用保持功能明确：

```text
@图片1 作为角色形象参考
@图片2 作为场景光影参考
@视频1 参考动作节奏和运镜衔接
@音频1 作为对白音色、环境声或动作音效参考
```

`@音频1` 默认不是音乐。只有用户明确提供音乐或要求音乐时，才把音频写成音乐参考。

## 特殊任务

### 视频延长

- 明确写“将 @视频1 延长约 X 秒”
- `镜头1` 必须与原视频最后状态无缝衔接
- 保持主体、光影、动作惯性、声音环境一致
- 新增动作要有自然起承转合

### 视频编辑

- 先写保留内容
- 再写修改点
- 最后写新动作或新结果
- 编辑后的镜头仍按 `镜头N` 四要素顺序描述

### 导演感转译

不要机械复述导演名。把用户说的“某导演感”翻译成构图、节奏、景别、机位、运镜、剪辑动机和声音处理。

## 生成前检查

输出前快速自检：

- 是否使用 `镜头1 / 镜头2`，而不是精确秒数切段
- 单条是否在 4-15 秒规划内，镜头数是否在 1-8
- 每个镜头是否按“四要素顺序”写完整
- 画幅与景别是否匹配
- 组内相邻镜头是否有剪辑动机
- 跨分镜组是否有空间、动作、声音、情绪或道具衔接
- 是否默认不写音乐
- prompt 正文是否没有 `【约束条件】`
