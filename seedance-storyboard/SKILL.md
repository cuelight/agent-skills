# Seedance 2.0 分镜 Skill

本 skill 的目标不是“把用户带进一轮很长的问卷”，而是先基于现有信息快速产出一版高质量首稿 prompt，再在必要时补 1-3 个关键问题。

适用场景：

- 用户只有一句想法，想快速变成 Seedance prompt
- 用户已经有剧情、角色或素材，希望整理成可直接投喂的视频脚本
- 用户想控制景别、运镜、人物关系、人物站位、画幅适配、情绪或导演感
- 用户要做视频延长、编辑、动作复刻、情绪戏、广告片段等

## 默认工作方式

默认采用“首稿优先”模式：

1. 先提取用户已提供的关键信息
2. 直接组织一版完整 prompt 首稿
3. 若缺少关键事实，再追问 1-3 个最小必要问题
4. 最后补一句简短优化建议

不要默认进入完整问卷流程。只有当以下关键信息缺失到足以影响执行时，才允许提问：

- 时长（4-15 秒）
- 主体是谁、在做什么
- 是否有参考素材可用
- 输出比例（用户没给且场景类型不足以判断默认时；详见「画幅适配」节）

如果用户已经给出足够信息，直接生成，不要再逐项盘问“风格、镜头、声音、转场”等。

## 输出格式

默认输出包含两部分：

1. `Prompt Draft`
   直接给出一版可复制使用的 Seedance prompt
2. `Optional Tweaks`
   用 2-4 条短建议说明还能如何继续优化，例如“补角色参考图”“把中段改成内反打”“把结尾收成 close-up”

只有在信息不足时，才先提出极少量问题；问题应短、明确、直接影响 prompt 质量。

## Prompt 组织顺序

输出的 prompt 默认按下面顺序组织：

1. 整体风格、时长、画幅、情绪基调（画幅先于景别 / 运镜 / 站位决定，并向下贯穿这些决策——见「画幅适配」节）
2. 时间轴分镜
3. 声音设计
4. 参考素材说明

每个时间段优先包含以下要素：

- 环境或空间锚点
- 景别
- 机位
- 运镜
- 主体动作
- 情绪表演
- 环境变化
- 声音设计

标准子分镜链条：

```text
景别 + 机位 + 运镜 + 主体动作 + 情绪表演 + 环境变化 + 声音设计
```

这 7 个要素可以写成一句自然镜头描述，不要机械列标签。`static shot` / 固定镜头是合法运镜，但仍要写清机位关系。情绪必须落到可见表演，例如眼神、呼吸、肩背、嘴角、手指、停顿；环境变化必须落到镜头内变化，例如灯光闪烁、门板晃动、雨水加重、尘土扬起、热气散开。

避免只写“镜头运动 + 画面 + 动作 + 特效”这种过于空泛的结构。

## 镜头选择规则

优先把镜头语言写成可执行的拍法，而不是抽象形容词。

### 景别选择

- `wide shot`：用于建立空间、交代人物站位、展示环境压力
- `medium shot`：默认主叙事镜头，最适合动作、互动和稳定阅读
- `close-up`：用于情绪落点、重要细节、心理变化
- `extreme close-up`：只在需要压迫感、危险细节或关键物件时使用

默认策略：

- 信息不多时，优先 `medium shot`
- 开场建立环境时，先给 `wide shot`
- 结尾情绪落点时，优先收在 `close-up`

### 运镜选择

- `static shot`：默认稳定、安全，适合对白、广告、情绪压抑场景
- `slow push in`：适合情绪升温、发现异常、心理压迫
- `pull out`：适合失落、收束、把人物放回环境
- `tracking shot`：适合行走、入场、追逐、连续动作
- `handheld`：适合冲突、危机、混乱，但只写“轻微手持感”或“纪实手持”，不要默认乱晃
- `orbit shot` / `arc shot`：只用于情绪顶点、告白、对峙、造型展示

默认不要在一个短视频里同时叠加多个复杂运镜方向。

### 机位选择

- `eye-level` / 平视：默认叙事机位，适合关系清楚、表演可读
- `low angle` / 低机位：适合压迫、权力、登场气场
- `high angle` / 高机位：适合脆弱、被围困、空间压迫
- `over-the-shoulder` / 过肩：适合对白、试探、关系张力
- `POV` / 主观视角：只在发现、窥视、恐惧、代入感强时使用

不要只写“镜头拍某人”。至少写清景别、机位和运镜，例如 `medium close-up, eye-level static shot`。

## 人物关系规则

当场景里有两人以上互动时，优先把“人物关系”写清楚，而不是只写动作。

- `two shot`：适合建立关系、同框对话、亲密或对立状态
- `over-the-shoulder`：适合稳定对白和关系张力
- 正反打：适合信息清楚的对话推进
- `外反打`：优先用于平稳沟通、叙事说明、常规对话
- `内反打`：优先用于逼问、争吵、暧昧、情绪升高
- `reaction shot`：当“听的人比说的人更有戏”时，优先拍反应而不是继续拍说话者

如果用户只说“两个人对话”，默认不要把全段都写成不停切镜。先用 `two shot` 或 `外反打` 建立关系，再在情绪升级处切入 `内反打` 或 `close-up`。

## 人物站位策略

景别和运镜决定“怎么看”，站位决定“看到什么关系状态”。短视频首稿优先写**粗粒度站位**，不要写画面坐标或朝向角度。

写站位时优先确定 5 件事：

- **距离**：贴近 / 一臂距离 / 数步之外 / 远距空间——直接反映关系紧张度
- **前后景**：前景压人 / 同一深度 / 后景孤立——反映权力或情绪状态
- **是否同框**：双人同框 vs 切单人——同框建立关系，切单人撕裂关系
- **入场方向**：从画面深处走来 / 从侧面切入 / 已在画面内——控制节奏与仪式感
- **走位变化**：靠近 / 远离 / 绕行 / 停滞——情绪转折的物理外化

默认值：

- 普通双人对话：一步距离，同一深度，双人同框，靠 `two shot` / `外反打` 收容
- 紧张对峙：数步之外，同一深度，保持同框，避免立刻切单人
- 权力压迫：前景压人 + 后景孤立，用 `low angle` 或 `over-the-shoulder` 强化
- 暧昧关系：近距离同框，但视线不直接对视
- 人物登场 / 仪式感：从画面深处或画面外缓步走入，配合 `slow-motion tracking shot`
- 多人场景：默认不超过 3 人有戏，多人用群像 + 主角前景，避免每个人都给镜头

避免：

- 写“角色 A 站在画面 1/3 处，朝向 45 度”——过细的坐标 / 角度模型反而失控
- 写“两个人对话”不带任何站位 / 距离信息——模型会任意安排，常见前后乱叠
- 复杂走位 + 复杂运镜同时叠加（例如“绕圈走 + 360 度环绕镜头”）——动作逻辑会崩

## 导演风格流派映射

当用户提到导演、电影感、作者风格时，不要机械复述导演名字，而要把它翻译成可执行镜头语言。

### 古典叙事流派

适用：剧情片、人物关系戏、短剧主线

写法重点：

- 清楚的空间建立
- 稳定的关系镜头
- `wide shot -> medium shot -> close-up` 的叙事递进

### 精准控制流派

适用：悬疑、制度压迫、控制感场景

写法重点：

- 对称构图
- 稳定机位
- 克制的 `slow push in`
- 冷静、秩序化空间

### 纪实手持流派

适用：冲突、危机、追逐、现场感

写法重点：

- 近距离跟随
- 轻微手持感
- 呼吸式不稳定
- 镜头贴近角色身体状态

### 长镜头调度流派

适用：空间漫游、仪式感、连续动作推进

写法重点：

- `long take`
- 连续走位
- 前景/中景/后景同时活动
- 少切镜，多调度

### 情绪化碎片流派

适用：都市夜景、暧昧、孤独、回忆感

写法重点：

- 遮挡构图
- 反射、霓虹、慢镜、失焦
- 情绪停顿而非纯信息推进

### 类型化强风格流派

适用：犯罪、黑色幽默、视觉包装强的片段

写法重点：

- 高设计感构图
- 强节奏切换
- 明显的空间几何和作者标签

使用原则：

- 先写流派特征，再少量提导演名
- 能不写导演名时，优先只写镜头语言特征
- 如果用户说“像某导演”，应拆成：构图、节奏、机位、运镜、情绪目标

## 短时长约束

Seedance 常见时长只有 4-15 秒，因此镜头复杂度必须受控。

- 4-6 秒：1-2 个主要信息点，优先单一动作或单一情绪转折
- 7-10 秒：2-3 个节拍，可建立 -> 变化 -> 收束
- 11-15 秒：允许更完整的小叙事弧线，但仍要克制切镜数量

默认限制：

- 不要在短时长里堆过多切镜
- 不要同时要求复杂运镜、复杂调度、复杂特效、密集对白
- 动作戏优先保证动作可读性，不要过分追求镜头花样
- 情绪戏优先保证表情和关系，不要把每一秒都写满

## 画幅适配

画幅不是装饰，它直接改变景别、运镜、站位的可执行性。同一个想法在 16:9 和 9:16 下的最优拍法常常不同。

### 横屏（16:9 / 2.35:1）

- **擅长**：广角空间建立、群像、横向运动、对称构图、风景与场景叙事
- **景别**：`wide shot` 与 `medium shot` 自然好拍；`close-up` 注意左右留白与负空间
- **运镜**：`horizontal pan` / `tracking shot` 横移自然；`orbit` 弧度可以更大
- **站位**：双人同框非常容易，一字排开 / 三角站位都可承载；多人 4-5 人仍可读
- **典型场景**：动作戏、风景片、广告展示、会议 / 群像戏、史诗感叙事

### 竖屏（9:16）

- **擅长**：单人特写、纵向运动、近景人物、强情绪戏、社媒爽剧
- **景别**：`close-up` 几乎默认压满画面，情绪强度天然更高；`wide shot` 难建立横向空间，建议改成“前景遮挡 + 中景人物”
- **运镜**：`slow push in` / `vertical tracking` / `tilt` 比横向运镜更稳；过分横移会丢主体
- **站位**：双人同框需要“近距离 + 一前一后”或“侧面贴近”，避免左右并排导致两人都被裁切；多人最多 2-3 人，超过用 `reaction shot` 拆开
- **典型场景**：短剧 / 心理戏 / 单人独白 / 都市情绪 / 时尚展示

### 决策原则

- 用户给了画幅就尊重画幅，不要在 9:16 里硬塞“群像横移”这种横屏专属拍法
- 用户没给画幅时，按场景类型默认：动作 / 广告 / 风景 → 16:9；短剧 / 单人情绪 / 社媒 → 9:16
- 同一个想法可以提示用户“这版按 9:16 拍，如果改横屏 16:9，会改为 [具体差异]”——只在用户没给且影响很大时主动提

## 多模态素材组织规则

当用户给出素材时，用最直接、最有功能性的方式写进 prompt。

- 图片：优先用于角色形象、服装、场景、首帧、尾帧
- 视频：优先用于动作节奏、运镜参考、特效参考、空间运动方式
- 音频：优先用于配乐节奏、音色、环境声或对白参考

引用素材时保持明确：

```text
@图片1 作为角色形象参考
@图片2 作为首帧
@视频1 参考运镜和动作节奏
@音频1 用于配乐氛围参考
```

不要把所有素材都塞进 prompt。优先引用对结果影响最大的素材。

## 特殊任务处理

### 视频延长

- 明确写“将@视频1延长 X 秒”
- 延续原视频风格、主体和动作逻辑
- 新增内容要强调“与前段无缝衔接”

### 视频编辑

- 先写保留什么
- 再写修改什么
- 最后写新剧情或新视觉结果

### 动作 / 运镜复刻

- 优先说明参考的是动作、运镜还是特效
- 不要笼统写“全部参考”，除非用户明确需要完整复刻

### 情绪戏

- 重点写关系、表情、镜头距离和节奏
- 少量声音、少量动作，比大段剧情更重要
- 情绪要写成可见表演：眼神收紧、呼吸压住、肩背绷紧、嘴角停住、手指发抖、短暂停顿
- 结尾可保留情绪尾韵：让环境音、动作余波或一次停顿承接，而不是只写“情绪复杂”

### 广告 / 展示片

- 重点写质感、构图、产品细节、节奏和落版
- 默认优先 `static shot`、`slow push in`、`close-up`

## 生成策略

当你拿到用户输入时，按下面顺序思考：

1. 当前任务属于叙事、动作、情绪、广告、延长还是编辑
2. 画幅是否已知；未知时按场景类型默认（动作 / 广告 / 风景 → 16:9；短剧 / 单人情绪 / 社媒 → 9:16），画幅决定后续景别 / 运镜 / 站位的可选项
3. 当前时长能承载多少信息点
4. 这一段最需要观众看到的是空间、动作、关系还是情绪
5. 应该优先使用哪种景别、机位和运镜
6. 是否需要关系镜头、站位策略或流派特征
7. 是否需要补问关键事实；如果不是必须，就直接生成首稿
8. 声音设计：决定每条声音的类型（dialogue / narration / internal_monologue / voice_over / off_screen / ambient_voice），并按"单句 3-12 字 / 单镜头 ≤ 8 秒 / 连续对白 ≤ 3 轮"裁剪长度
9. 检查每条子分镜是否覆盖七要素：景别、机位、运镜、主体动作、情绪表演、环境变化、声音设计

## 推荐输出骨架

```text
Prompt Draft

[整体风格 + 时长 + 比例 + 基调]

0-X秒：景别 / 机位 / 运镜 / 主体动作 / 情绪表演 / 环境变化 / 声音设计
X-Y秒：景别 / 机位 / 运镜 / 主体动作 / 情绪表演 / 环境变化 / 声音设计
...

【声音】配乐 / 环境音 / 对白
【参考素材】@图片1 ...，@视频1 ...

- ...
- ...
```

## 对白与声音设计

写【声音】段时，对白 / 旁白必须从下列 6 类中**显式选一类**，模型才能正确决定是否生成嘴型、是否把声源放到画外：

- `dialogue`（普通对白，画内、有嘴型）
- `narration`（旁白，推进剧情）
- `internal_monologue`（内心独白，画内、无嘴型）
- `voice_over`（回忆 / 跨时空画外音）
- `off_screen`（场外说话、电话另一端）
- `ambient_voice`（群众 / 环境氛围）

长度规范：单句对白 **3–12 字**，单镜头总语音 **≤ 8 秒**，连续对白 ≤ 3 轮。

工业级最稳模板：**视觉动作 + 一句对白 + 一句情绪旁白**。复杂动作戏避免长对白，改用内心独白或画外音。

详细分类、各类型推荐写法、反例与 JSON DSL 字段标准见 [references/dialogue-narration.md](references/dialogue-narration.md)。

## 追问规则

仅在以下情况追问，而且控制在 1-3 个问题：

- 时长完全缺失且无法合理推断
- 主体是谁、在做什么不明确
- 是否有参考素材会明显改变 prompt 结构
- 画幅没给，且场景类型不足以判断默认（例如想法适合两种画幅且差异显著）；能按"画幅适配"节默认时不要追问

不要追问那些可以合理默认的内容，例如：

- “是不是要电影感”
- “要不要特效”
- “色调是暖色还是冷色”
- “要不要配乐”

这些都应先做专业默认，然后通过首稿体现。
